Active regression — the LLC worker is losing data

This deserves its own tab because it changed character since the last sweep: it's no longer just "stalled," it's actively pulling rows into a dead state.

Queue state2026-05-30last sweepnowdirection
dia queued1,2581,2081,044draining ↓

gov queued657607460draining ↓
gov in_progress (stuck)050200climbing ↑
completions000none
Mechanism. Each 30-min tick marks a fresh batch in_progress, calls the lookup, gets no_handler_configured, and returns without resetting the rows. The backoff-to-deferred path never fires because it's gated behind a resolved lookup. So queued rows migrate to a permanent in_progress limbo and never come back. At this rate both queues fully convert to dead rows within days.
Two-part fix (the recommended next build): (1) a small reset so a tick that bails on no_handler_configured returns rows to queued (or moves them to deferred after the cap) instead of stranding them in_progress; (2) configure an actual research handler (free SOS-direct adapter or OpenCorporates key) so lookups resolve and owner coverage finally climbs.
