audit(item-6/phase-b2): NBA queue completeness weighting

Phase B-2 of Item #6. Builds directly on Phase B-1 (persisted
completeness column) — that one set the table; this one puts it to
work.

v_next_best_action on both dia + gov now multiplies the per-gap-type
gap_value by a completeness factor sourced from the persisted
properties.completeness_band column.

Multipliers:
  excellent (90+)  → 1.50x   closing this gap finishes the underwriting
  good      (70-89)→ 1.25x   closing this gap brings it close to done
  fair      (40-69)→ 1.00x   neutral (unchanged from previous behavior)
  poor      (<40)  → 0.80x   many other gaps remain; less leverage
  NULL band         → 1.00x   defensive

API surface:
  • gap_value         — now the WEIGHTED value (rank reflects this)
  • raw_gap_value     — pre-weighting (preserved for transparency)
  • completeness_band — exposed for UI hints
  • completeness_score— precise score for sort tiebreaks
  • /api/admin?_route=next-best-action cross-domain merge sorts by
    gap_value DESC — picks up the weighting with zero code change.
  • NBA Home rail (renders gap_value as the deal value) continues
    to work.

Both migrations already applied via MCP at 2026-05-17.
