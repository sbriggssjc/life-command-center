# Copilot Action Registry

## Purpose
Define the authoritative inventory of Copilot actions for the brokerage operating system, including ownership, routing, risk, confirmation policy, and target Microsoft surfaces.

## Outline
1. Action metadata schema reference
2. Action catalog by domain and category
3. Ownership and routing (`LCC`, `GovernmentProject`, `DialysisProject`)
4. Risk tiers and confirmation requirements
5. Idempotency and audit requirements
6. Inputs/outputs contracts per action
7. Surface mappings (Teams, Outlook, Power Automate, Planner, Excel, etc.)
8. Versioning and deprecation policy
