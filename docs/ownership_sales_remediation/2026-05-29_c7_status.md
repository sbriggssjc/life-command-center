# Ownership & Sales Remediation — C7 status (SOS adapter framework)

Date: 2026-05-29. The last open plan item. Assessed feasibility in this environment.

## Outcome: C7 framework already scaffolded; per-state adapters are a **workstation/data-pipeline task** (cannot be developed or verified from this environment)

### What already exists (no further framework code needed)

`api/_shared/llc-research.js` already implements the adapter framework:
- `lookupLlc({name, state})` orchestrator — checks a per-state adapter **first**, falls through to OpenCorporates (feature-flagged on `OPENCORPORATES_API_KEY`).
- `SOS_DIRECT_ADAPTERS` registry (state code → adapter, uniform `lookupLlc` shape, `source:'sos_<state>'`). Currently empty (FL entry commented, pending its mirror+parser).
- Worker `llc-research-tick` (admin.js) drains `llc_research_queue` and writes results into `recorded_owners`; manual `POST /api/sos-writeback` is the 45-state fallback. Both routes live.

The code's own build strategy + guardrail (verbatim): bulk-download states (FL Sunbiz data files) preferred → free search/API states → else OpenCorporates / sidebar-manual; and **"Adapters MUST be verified against the live/source format before enabling (return adapter_pending until then)."**

### Why C7 can't be built/verified here

- **All five target SOS endpoints return HTTP 403 from this execution environment** — TX (`mycpa.cpa.state.tx.us`), FL (`search.sunbiz.org`), CA (`bizfileonline.sos.ca.gov`), GA (`ecorp.sos.ga.gov`), NC (`sosnc.gov`). Datacenter-IP / network-policy block. I can neither scrape nor verify an adapter against its live source from here, and the code contract forbids enabling an unverified adapter.
- The **preferred FL path is a bulk-data mirror** (download Sunbiz's corporate data files → parse → local match), which is a data-pipeline task needing source access + storage + a matching job — not something this stateless, network-restricted environment can stand up or test.

### Current impact = low / no urgency

- gov `llc_research_queue`: **624 rows** queued (plan-era estimate was ~1,696 gov+dia). Rows **persist harmlessly** — the worker no-ops on `adapter_pending` and (without `OPENCORPORATES_API_KEY`) leaves rows queued; they resume the moment an adapter or the key lands. No data loss, no errors.
- C7 is explicitly **lowest-priority / not on the critical path**, and the standing preference is free SOS-direct over paid OpenCorporates — which specifically requires the iterative, browser/residential-IP, captcha-aware development a real workstation provides.

## Recommendation

Build C7 incrementally **from a workstation/browser context with SOS access**, one state at a time, FL-bulk-mirror first per the code's documented strategy, each adapter verified live before its registry entry is enabled. It is not a remote-agent deliverable. Until then: queue persists; OpenCorporates remains the flagged fallback; `/api/sos-writeback` covers manual sidebar capture.
