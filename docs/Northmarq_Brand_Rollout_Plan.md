# Northmarq Brand Rollout Plan — Client-Facing Outputs

## Brand Identity Reference

Extracted from northmarq.com live site (April 2026):

| Token | Hex | Usage |
|-------|-----|-------|
| Primary Blue | `#003DA5` | Links, CTA backgrounds, section headers, accents |
| Deep Navy | `#001159` | Report headers, footer bars, dark headings |
| Light Blue | `#62B5E5` | Secondary accent, chart highlights |
| Blue Tint | `#E0E8F4` | Light backgrounds, methodology boxes |
| Warm White | `#FAF9F5` | Alternating table rows, KPI card backgrounds |
| Body Text | `#3D4A54` | Primary copy color |
| Dark Text | `#191919` | Headings, bold values |
| Muted | `#6A748C` | Secondary labels, captions |
| Border | `#D8DFDF` | Table borders, dividers |

**Typography:**
- Headings: `futura-pt` (Futura PT) — fallback: Trebuchet MS, Arial
- Body: `Open Sans` — fallback: Segoe UI, system-ui

**Logo:** `https://www.northmarq.com/themes/custom/northmarq/logo.svg`

**Constants in code:** `NMQ_BRAND` object in `detail.js` (line ~3132)

## Implementation Notes

- The `NMQ_BRAND` constant object in `detail.js` is the single source of truth for frontend brand values
- For Python/openpyxl outputs (BOV, exhibits), create a parallel `NMQ_COLORS` dict in the skill references
- Logo SVG loads from northmarq.com CDN; fallback text "NORTHMARQ" renders if SVG fails (handles offline/print scenarios)
- Google Fonts `Open Sans` is loaded via CDN in export HTML; works in all modern browsers and PDF renderers
- All branded exports include `@media print` rules for clean PDF generation via browser Print dialog
